## 프로젝트 선정 이유
1. 요약은 상대적으로 쉬운 태스트케 속하는 편 (주어진 텍스트만 사용, 외부 지식 및 상식 필요 X)
2. OpenAI도 모델 개발 과정을 요약에서 출발해서 대화로 이동함
3. 태스크를 쉽게 설정하여 실제 AI 서비스를 end-to-end로 구현하는데 focus

## 프로젝트 목표
- AI 서비스 개발 과정을 end-to-end로 경험해보는 것

## 문제 조건 설정 이유
- 서비스 품질과 관련된 내용
- 요약 품질이 저하 될 수 있음
- LLM이 감당 할 수 없는 입력이 들어 올 수 있음

### 문제 조건 1
- 숙소 유형에 따라 입력 데이터의 기간 변경 필요
  - 모텔은 최근 3개월 데이터 사용
  - 호텔/펜션/게스트하우스는 최근 6개월 후기 데이터 사용
### 문제 조건 2
- 후기 글이 너무 적을 경우 요약 결과 없음 처리 / 품질 이슈 가능성
  - 후기는 3개 이상
  - 후기 글의 합이 90자 이상
### 문제 조건 3
- 후기가 많을 경우 (수치화 X)
- 모델의 Context Window가 감당 가능한 후기 수
- gpt-3.5-turbo-0125는 최대 16,385 토큰 입력 가능
- 비용을 고려한 적절한 후기 수
  - 후기가 많을 경우, 최근 작성 기준으로 우선 요약
